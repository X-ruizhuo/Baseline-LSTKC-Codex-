# LSTKC++ 融合改造说明（16G 显存约束版）

## 1. 背景与目标

本次改造以现有 `Bi-C2R` 训练框架为基础，参考 `LSTKC++` 的“长短期知识分解与巩固”思想，在**不重写主干结构**的前提下完成工程级融合，并满足单卡 **16G 显存**可训练的部署约束。

核心目标：

- 保留原有持续学习主流程与评测接口，降低迁移风险；
- 引入低成本可复用的显存优化机制，确保可运行性；
- 在“稳定性（抗遗忘）-可塑性（新任务学习）”之间保持可控平衡。

---

## 2. 两个方法的定义

为便于对比，本文将两种方法定义如下：

- **方法A（原始流程）**：项目改造前的 `Bi-C2R` 默认训练流程（FP32、单步反传、batch=64）。
- **方法B（融合流程）**：本次改造后的 `Bi-C2R + LSTKC++风格低侵入融合`（AMP + 梯度累积 + 16G默认参数）。

> 说明：本次属于“Loss/Trainer 级融合”，即在训练器层面对长期/短期知识约束进行保留和强化，不改变主干网络的核心拓扑。

---

## 3. 创新内容（本次改造）

### 创新点1：显存感知的持续学习训练范式

在持续学习场景下引入“**AMP 混合精度 + 梯度累积**”组合，并作为默认训练策略落地到主入口。

- AMP：通过半精度计算降低激活显存占用；
- 梯度累积：在较小物理 batch 下模拟更大等效 batch，减少优化不稳定；
- 两者协同：在 16G 环境保持吞吐与收敛稳定性的工程折中。

### 创新点2：与知识巩固损失的兼容式设计

原有抗遗忘、变换约束、判别蒸馏等损失项均保留，通过训练器统一进入 autocast 计算域，避免“只省显存但破坏损失结构”的常见问题。

这使得本次优化不是单纯“减配”，而是**面向持续学习目标函数的一体化适配**。

### 创新点3：参数与脚本的可复现实验落地

不仅修改核心代码，还同步更新运行脚本与说明，形成可直接执行的 16G 配置闭环：

- 默认 `batch-size` 从 64 调整到 32；
- 默认 `workers` 从 8 调整到 4；
- 默认开启 `amp`；
- 默认 `grad-accum-steps=2`；
- `run1.sh` / `run2.sh` 与 README 同步。

---

## 4. 两个方法的差异

## 4.1 训练机制差异

- 方法A：FP32 全精度前向与反向，单 iter 一次参数更新；
- 方法B：autocast 混合精度 + GradScaler，按累积步数更新参数。

## 4.2 显存与吞吐差异

- 方法A：高 batch 下显存峰值高，16G 卡容易触发 OOM；
- 方法B：低物理 batch + 混合精度显著降低峰值显存，可在 16G 单卡稳定运行。

## 4.3 持续学习稳定性差异

- 方法A：若直接降 batch，可能导致梯度噪声上升，影响抗遗忘项和迁移项平衡；
- 方法B：通过梯度累积补偿小 batch 带来的统计波动，保持损失项协同训练的稳定性。

## 4.4 工程可迁移性差异

- 方法A：参数硬编码偏大卡（如 A40/24G+）场景；
- 方法B：默认参数面向中等资源服务器，迁移成本更低，复现实验门槛更低。

---

## 5. 创新的原因（为什么这样设计）

### 原因1：硬件约束是真实主约束

在科研和工程部署中，16G 显卡是高频资源上限。如果方法只能在大显存卡上复现，算法可用性会显著下降。因此需要优先做“可训练化改造”。

### 原因2：持续学习对优化稳定性更敏感

相比单任务训练，持续学习包含多重损失（分类、度量、抗遗忘、变换、蒸馏等）。简单粗暴降 batch 可能引起损失权重失衡。梯度累积是更稳妥的工程手段。

### 原因3：低侵入融合可控、可回退

本次在 Trainer 和入口层改造，不重构 backbone/head。这样可：

- 快速验证收益；
- 减少结构性回归风险；
- 便于后续继续向 Model 级融合演进。

## 7. 预期收益与边界

### 预期收益

- 单卡 16G 场景下训练可用性显著提升；
- 持续学习损失框架保持完整；
- 复现实验与迁移部署更轻量。

### 当前边界

- 本次未引入新的网络分支结构（仍属 Trainer 级融合）；
- 极端长序列任务下仍需进一步做显存与速度联合优化（如 checkpointing、分段缓存策略）。

---

## 8. 后续可扩展方向

- 增加 OOM 自适应降级（自动下调 batch 或提升累积步）；
- 按阶段自适应权重调度（动态平衡抗遗忘与新知识学习）；
- 进一步推进到 Model 级 long/short 显式分支融合，提升上限性能。
